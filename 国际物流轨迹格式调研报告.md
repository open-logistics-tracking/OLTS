# 国际物流轨迹格式调研报告

**International Logistics Tracking Format Research Report**

> 版本：v1.0 | 日期：2026-05-13 | 编制：基于公开资料整理

---

## 一、执行摘要 | Executive Summary

本报告系统调研了国际物流轨迹的主流格式和标准，覆盖国际快递巨头（DHL、FedEx、UPS、USPS）、国际邮政、跨境电商物流节点、海关清关状态等。核心目的是**为OLTS（Open Logistics Tracking Schema）设计提供国际兼容性参考**，避免后续支持国际物流单号时需要大规模改版。

### 核心发现

| 维度 | 国际段特征 | 与国内差异 |
|:---|:---|:---|
| **时间格式** | 强制ISO 8601 + 时区偏移（如`2024-06-03T11:38:00+07:00`）| 国内大多无时区 |
| **地址编码** | 国家代码（ISO 3166-1 alpha-2）+ 邮政编码 | 国内用行政区划码 |
| **清关节点** | 海关申报、查验、放行、扣关、缴税等专属状态 | 国内无此环节 |
| **多式联运** | 航空/海运/陆运切换，含航班/船期信息 | 国内以陆运为主 |
| **时区处理** | 必须支持多时区转换 | 国内单一时区 |
| **语言本地化** | 状态描述需多语言支持 | 国内中文为主 |

### 关键结论

> **OLTS设计必须预留国际扩展字段**，核心数据模型（事件类型、状态码、地址结构）需要兼容国际物流的特有节点，否则后续改版成本极高。

## 二、国际快递巨头API格式 | Global Express Carriers

### 2.1 DHL Shipment Tracking - Unified API

| 维度 | 详情 |
|:---|:---|
| **接口URL** | `https://api-eu.dhl.com/track/shipments` |
| **认证方式** | API Key（`DHL-API-Key` Header） |
| **请求方式** | GET |
| **支持业务** | DHL Express、DHL Freight、DHL eCommerce、DHL Parcel（德国/英国/荷兰）、Post & Parcel Germany |
| **速率限制** | 250次/天（开发级），max 1次/5秒 |

#### 请求参数

| 参数名 | 类型 | 必填 | 说明 |
|:---|:---|:---:|:---|
| `trackingNumber` | string | ✅ | 运单号 |
| `service` | string | ❌ | 服务类型：`express`/`svb`/`parcel-de`/`ecommerce`/`freight`等 |
| `originCountryCode` | string | ❌ | 发货国家代码（如`NZ`） |
| `requesterCountryCode` | string | ❌ | 请求者国家代码（如`GB`） |
| `recipientPostalCode` | string | ❌ | 收件人邮编（德国/荷兰专用） |
| `language` | string | ❌ | 返回消息语言 |

#### 核心响应结构

```json
{
  "shipmentId": "string",
  "trackingNumber": "string",
  "status": {
    "statusCode": "transit",
    "status": "具体状态代码",
    "description": "状态描述",
    "remark": "次要描述",
    "timestamp": "2024-06-03T11:38:00+07:00"
  },
  "estimatedTimeOfDelivery": "2024-06-05",
  "service": "express",
  "origin": { "address": { "countryCode": "CN" } },
  "destination": { "address": { "countryCode": "DE" } },
  "events": [
    {
      "timestamp": "2024-06-03T11:38:00+07:00",
      "statusCode": "transit",
      "status": "状态代码",
      "description": "事件描述",
      "remark": "备注",
      "location": {
        "address": {
          "addressLocality": "SHANGHAI - Shanghai - CHINA",
          "countryCode": "CN"
        }
      }
    }
  ],
  "pieces": [
    {
      "pieceId": "piece1",
      "events": [ /* 同events结构 */ ]
    }
  ]
}
```

#### DHL轨迹字段映射（→ OLTS）

| OLTS标准字段 | DHL字段 | 说明 |
|:---|:---|:---|
| `eventTime` | `events[].timestamp` | ISO 8601含时区偏移 |
| `description` | `events[].description` | 多语言支持 |
| `status` | `events[].statusCode` | 英文编码（如`transit`/`delivered`） |
| `location.name` | `events[].location.address.addressLocality` | 城市-州-国家格式 |
| `location.countryCode` | `events[].location.address.countryCode` | ISO 3166-1 alpha-2 |
| `remark` | `events[].remark` | 次要描述 |
| `pieceId` | `pieces[].pieceId` | 子包裹ID |

#### DHL特色设计

| 特色 | 说明 | 对OLTS的启示 |
|:---|:---|:---|
| **JSON-LD兼容** | 响应语义化，支持关联数据 | OLTS建议采用JSON-LD或Schema.org词汇 |
| **分层状态** | `status.statusCode`（通用）+ `status.status`（业务线特定） | OLTS状态码应分层：通用+承运商特定 |
| **Piece级跟踪** | 支持子包裹（piece）独立跟踪 | OLTS需支持`pieceId`字段 |
| **时区强制** | 2024.9.14起Express强制含时区偏移 | OLTS时间字段必须含时区 |
| **多服务适配** | 同一API适配Express/Freight/eCommerce/Parcel | OLTS应通过`serviceType`区分 |
| **邮编验证** | 部分详细数据需邮编验证后返回 | OLTS可定义`requiredCredentials`字段 |

### 2.2 UPS Tracking API

| 维度 | 详情 |
|:---|:---|
| **接口URL** | `https://onlinetools.ups.com/api/track/v1/details/{inquiryNumber}` |
| **认证方式** | OAuth 2.0 Client Credentials |
| **请求方式** | GET |
| **数据保留期** | 120天 |

#### 认证签名

| 项目 | 说明 |
|:---|:---|
| **Token Endpoint** | `https://onlinetools.ups.com/security/v1/oauth/token` |
| **方法** | `client_secret_basic` |
| **Scope** | `{}`（无特定范围） |

#### 请求参数

| 参数名 | 位置 | 必填 | 说明 |
|:---|:---|:---:|:---|
| `inquiryNumber` | Path | ✅ | 运单号（7-34位） |
| `transId` | Header | ✅ | 请求唯一标识 |
| `transactionSrc` | Header | ✅ | 调用来源 |
| `locale` | Query | ❌ | 语言国别（如`en_US`） |
| `returnSignature` | Query | ❌ | 返回签收签名（base64） |
| `returnMilestones` | Query | ❌ | 返回里程碑信息 |
| `returnPOD` | Query | ❌ | 返回送达证明 |

#### 核心响应结构

```json
{
  "trackResponse": {
    "shipment": [{
      "inquiryNumber": "1Z999AA10123456784",
      "package": [{
        "trackingNumber": "1Z999AA10123456784",
        "currentStatus": { "description": "Delivered" },
        "statusCode": "011",
        "statusDescription": "Delivered",
        "activity": [
          {
            "date": "20210210",
            "time": "071356",
            "gmtDate": "20210210",
            "gmtTime": "74700",
            "gmtOffset": "-05:00",
            "location": {
              "address": {
                "city": "Wayne",
                "stateProvince": "NJ",
                "postalCode": "07470",
                "country": "US"
              }
            },
            "status": {
              "code": "SR",
              "description": "Your package was released by the customs agency.",
              "simplifiedTextDescription": "Delivered",
              "statusCode": "003",
              "type": "X"
            },
            "logicalScan": false
          }
        ],
        "milestones": [...],
        "deliveryDate": [{ "type": "DEL", "date": "20210210" }],
        "weight": { "unitOfMeasurement": "LBS", "weight": "5.5" },
        "service": { "description": "UPS Ground" }
      }]
    }]
  }
}
```

#### UPS轨迹字段映射（→ OLTS）

| OLTS标准字段 | UPS字段 | 说明 |
|:---|:---|:---|
| `eventTime` | `activity[].date` + `activity[].time` | 需拼接为完整时间，含gmtOffset |
| `description` | `activity[].status.description` | 支持本地化翻译 |
| `status` | `activity[].status.code` | 2字母代码（如`SR`/`PU`/`OD`） |
| `statusDetailCode` | `activity[].status.statusCode` | 3位数字代码（如`003`） |
| `location.city` | `activity[].location.address.city` | 城市 |
| `location.state` | `activity[].location.address.stateProvince` | 州/省 |
| `location.postalCode` | `activity[].location.address.postalCode` | 邮编 |
| `location.countryCode` | `activity[].location.address.country` | 国家 |
| `isLogicalEvent` | `activity[].logicalScan` | true=系统逻辑事件，false=物理扫描 |
| `weight` | `weight.weight` | 重量（含单位LBS/KGS） |

#### UPS特色设计

| 特色 | 说明 | 对OLTS的启示 |
|:---|:---|:---|
| **双时间系统** | 本地时间（date+time）+ GMT时间（gmtDate+gmtTime+gmtOffset） | OLTS应同时保留`localTime`和`utcTime` |
| **逻辑事件标识** | `logicalScan`区分物理扫描和系统逻辑事件 | OLTS需`isLogicalEvent`字段 |
| **里程碑体系** | 独立的`milestones`数组，按时间正序 | OLTS可增加`milestones`概念 |
| **送达日期类型** | `SDD`/`RDD`/`DEL`区分计划/改期/实际 | OLTS可定义`deliveryDateType` |
| **重量单位** | 支持`LBS`和`KGS` | OLTS重量字段需含`unit` |
| **敏感数据控制** | 签名/POD需包裹权限 | OLTS可定义`accessLevel` |
| **UCIX状态** | `O`(开放)/`C`(关闭)标识追踪状态 | OLTS可增加`trackingStatus` |

### 2.3 USPS Tracking API

| 维度 | 详情 |
|:---|:---|
| **接口URL** | `https://apis.usps.com/tracking/v3/tracking/{Tracking Number}` |
| **认证方式** | OAuth 2.0 Bearer Token |
| **请求方式** | GET |
| **查询模式** | `expand=summary`（摘要）或 `expand=DETAIL`（详细） |

#### 请求参数

| 参数名 | 位置 | 必填 | 说明 |
|:---|:---|:---:|:---|
| `Tracking Number` | Path | ✅ | USPS跟踪号码 |
| `expand` | Query | ✅ | `summary` 或 `DETAIL` |

```json

  "trackingNumber": "XXXXXXXXXXXXXXXXXXXX",
  "mailClass": "Priority Mail<SUP>&reg;</SUP>",
  "mailType": "DM",

  "originCity": "RICHMOND",
  "originState": "VA",
  "originZIP": "23227",
  "destinationCity": "CEDAR RAPIDS",
  "destinationState": "IA",

  "statusSummary": "USPS is now in possession of your item as of 7:31 am on August 2, 2023 in RICHMOND, VA 23227.",
  "trackingEvents": [
    {
      "eventType": "USPS in possession of item",
      "eventTimestamp": "2023-08-02T07:31:00Z",
      "eventCountry": null,
      "eventCity": "RICHMOND",
      "eventState": "VA",
      "eventZIP": "23227",
      "firm": null,
      "name": null,
      "authorizedAgent": "false",
      "eventCode": "03",
      "additionalProp": null
    }
  ]

```

#### USPS轨迹字段映射（→ OLTS）

| OLTS标准字段 | USPS字段 | 说明 |
|:---|:---|:---|
| `eventTime` | `trackingEvents[].eventTimestamp` | ISO 8601 UTC格式 |
| `description` | `trackingEvents[].eventType` | 英文描述 |
| `status` | `trackingEvents[].eventCode` | 2位数字代码 |
| `location.city` | `trackingEvents[].eventCity` | 城市 |
| `location.state` | `trackingEvents[].eventState` | 州 |
| `location.postalCode` | `trackingEvents[].eventZIP` | 邮编 |
| `location.countryCode` | `trackingEvents[].eventCountry` | 国家 |
| `isAuthorizedAgent` | `trackingEvents[].authorizedAgent` | 是否为授权代理 |

#### USPS特色设计

| 特色 | 说明 | 对OLTS的启示 |
|:---|:---|:---|
| **HTML实体编码** | 字段值含HTML标签（如`<SUP>&reg;</SUP>`） | OLTS字段应纯文本，避免HTML |
| **布尔值字符串化** | `authorizedAgent`: `"true"`/`"false"` | OLTS应规范布尔值类型 |
| **始发/目的地分离** | 顶层直接返回`originCity`/`destinationCity` | OLTS可增加`origin`/`destination`对象 |
| **状态分层** | `status`（描述）+ `statusCategory`（类别）+ `statusSummary`（摘要） | OLTS状态分层设计已验证 |
| **服务类型代码** | `serviceTypeCode`: `14`等 | OLTS可定义`serviceType` |
| **邮件类别** | `mailClass`: `Priority Mail` | OLTS可增加`serviceLevel` |

## 三、国际物流与国内物流的关键差异 | Key Differences

### 3.1 轨迹节点差异

| 节点类型 | 国内物流 | 国际物流 | 说明 |
|:---|:---|:---|:---|
| **揽收** | ✅ 快递员揽收 | ✅ 前端揽收/仓库收货 | 国际段可能在海外仓开始 |
| **转运中心** | ✅ 网点/转运中心 | ✅ 分拨中心/Hub | 命名不同 |
| **干线运输** | ✅ 陆运/航空 | ✅ **航空/海运/铁路** | 国际段多式联运 |
| **出口清关** | ❌ 无 | ✅ **报关/查验/放行** | 国际段特有 |
| **国际运输** | ❌ 无 | ✅ **航班起飞/到达** | 国际段特有 |
| **进口清关** | ❌ 无 | ✅ **海关申报/查验/缴税/放行** | 国际段特有 |
| **中转分拨** | ✅ 转运中心 | ✅ **海外中转仓** | 国际段可能有多次中转 |
| **尾程派送** | ✅ 快递员派送 | ✅ **当地快递派送** | 国际段由目的国快递完成 |
| **退件** | ✅ 退回发件人 | ✅ **跨境退件/销毁** | 国际退件成本极高 |

### 3.2 国际段特有轨迹状态

| 阶段 | 中文状态 | 英文状态 | 说明 |
|:---|:---|:---|:---|
| **出口清关** | 已报关 | Customs Declaration Filed | 向海关提交出口申报 |
| | 海关查验 | Customs Inspection | 海关检查货物 |
| | 出口放行 | Export Cleared / Released | 海关允许出口 |
| | 退关 | Customs Withdrawal | 撤销出口申报 |
| **国际运输** | 已离港 | Departed Origin Port | 离开起运国口岸 |
| | 航班起飞 | Flight Departed | 航空运输起飞 |
| | 航班到达 | Flight Arrived | 航空运输到达 |
| | 到港 | Arrived at Destination Port | 到达目的国口岸 |
| | 运输中 | In Transit / Transshipment | 中转运输中 |
| **进口清关** | 送交海关 | Presented to Customs | 物流商将货物移交海关 |
| | 海关申报 | Customs Entry Filed | 进口申报已提交 |
| | 等待缴税 | Awaiting Payment of Duties | 等待收件人缴税 |
| | 海关查验 | Customs Examination | 海关详细检查 |
| | 海关放行 | Released from Customs | 海关允许进口 |
| | 海关扣留 | Held in Customs / Seized | 被海关暂扣 |
| | 退运 | Return to Shipper (RTS) | 退回发件人 |
| **尾程** | 到达当地网点 | Arrived at Local Facility | 到达目的国派送网点 |
| | 派送中 | Out for Delivery | 当地快递员派送 |
| | 已签收 | Delivered | 目的国签收 |
| | 投递失败 | Delivery Attempted / Failed | 派送失败 |

### 3.3 数据格式差异

| 维度 | 国内物流 | 国际物流 | OLTS建议 |
|:---|:---|:---|:---|
| **时间格式** | `yyyy-MM-dd HH:mm:ss` | **ISO 8601 + 时区**（`2024-06-03T11:38:00+07:00`）| 统一为ISO 8601含时区 |
| **国家代码** | 无或`CN` | **ISO 3166-1 alpha-2**（`US`/`DE`/`JP`）| 必须含`countryCode` |
| **邮政编码** | 6位数字 | 各国不同（美国5位/英国字母数字/日本7位）| 定义为`postalCode`字符串 |
| **地址格式** | 省-市-区-街道 | **国家-州/省-城市-邮编-街道** | 分层地址结构 |
| **语言** | 中文 | **多语言**（需本地化翻译）| `description`支持多语言 |
| **重量单位** | kg（隐含）| **LBS/KGS** | 重量字段含`unit` |
| **货币** | 无 | **USD/EUR/JPY等** | 增加`currency`字段 |

## 四、跨境电商物流节点 | Cross-Border E-Commerce Logistics

### 4.1 八大核心节点

```
┌─────────────────────────────────────────────────────────────────┐
│                   跨境电商物流全链路节点                          │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 前端揽收 (First Mile Pickup)                                │
│     └─ 商家/海外仓 → 物流商收货                                  │
│                                                                 │
│  2. 库内操作 (Warehouse Operation)                              │
│     └─ 分拣、打包、贴标、称重                                    │
│                                                                 │
│  3. 出口通关 (Export Customs Clearance)                         │
│     └─ 报关 → 查验 → 放行 → 结关                                │
│                                                                 │
│  4. 干线运输 (Main Carriage / Line Haul)                        │
│     └─ 航空货运 / 海运 / 铁路 / 卡航                            │
│        └─ 航班号/船名/车次信息                                   │
│                                                                 │
│  5. 境外清关 (Import Customs Clearance)                         │
│     └─ 海关申报 → 查验 → 缴税 → 放行                            │
│                                                                 │
│  6. 中转分拨 (Transshipment / Hub Sorting)                      │
│     └─ 海外转运中心 → 当地分拨                                   │
│                                                                 │
│  7. 尾程派送 (Last Mile Delivery)                               │
│     └─ 当地快递商 → 末端派送                                     │
│                                                                 │
│  8. 退件处理 (Return Handling)                                  │
│     └─ 当地退件仓 → 跨境退回 / 销毁                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 各节点代表企业

| 节点 | 代表企业/平台 |
|:---|:---|
| 前端揽收 | 菜鸟裹裹、顺丰、京东快递、四通一达 |
| 库内操作 | 菜鸟保税仓、京东海外仓、顺丰国际仓、4PX递四方 |
| 出口通关 | 中国电子口岸、各地报关行 |
| 干线运输 | 航空公司（CA/CZ/MU）、船公司（COSCO/MAERSK）、中欧班列 |
| 境外清关 | 目的国海关（US CBP、EU TAXUD、Japan Customs） |
| 中转分拨 | DHL/FedEx/UPS Hub、菜鸟海外仓、4PX海外仓 |
| 尾程派送 | USPS、Royal Mail、DHL Parcel、当地快递商 |
| 退件处理 | 菜鸟退件仓、本地退件服务商 |

### 4.3 国际物流轨迹的特殊信息

| 信息类型 | 示例 | 说明 |
|:---|:---|:---|
| **航班信息** | `Flight CA1234 departed from PVG` | 航空运输需记录航班号 |
| **船期信息** | `Vessel COSCO SHANGHAI departed` | 海运需记录船名/航次 |
| **集装箱号** | `Container MSCU1234567` | 海运集装箱跟踪 |
| **MAWB/HAWB** | `Master AWB: 999-12345678` | 航空主/分运单号 |

### 6.1 数据模型扩展建议

#### 6.1.1 地址结构扩展

```json
{

    "address": {
      "countryCode": "CN",
      "countryName": "中国",
```

**关键变化**：
- `countryCode`: ISO 3166-1 alpha-2（必须）

- `eventTime`: 事件发生地本地时间（含时区偏移）

## 九、结论与建议 | Conclusions & Recommendations

### 9.1 核心结论

1. **国际物流轨迹格式与国内存在本质差异**：时区、国家代码、多语言、清关节点、多式联运信息是国内完全没有的维度。

2. **国际巨头（DHL/UPS/USPS）的设计更成熟**：
   - 强制ISO 8601 + 时区偏移
   - 分层地址结构（国家-州-市-邮编）
   - 多语言本地化支持
   - Piece-level子包裹跟踪
   - 逻辑事件/物理事件区分

3. **国际段特有节点必须预留**：清关（申报/查验/放行/扣关/缴税）、航空（航班起飞/到达）、海运（船舶离港/到港）、转运、退运等。

4. **IATA ONE Record是最值得借鉴的模型**：JSON-LD语义化、单一记录视图、OpenAPI 3.0接口、GitHub开源——这与OLTS的技术路线完全一致。

### 9.2 对OLTS v0.1的修订建议

| 修订项 | 原设计 | 修订后 |
|:---|:---|:---|
| **时间字段** | `eventTime: string` | `eventTime: string`（含时区）+ `eventTimeUTC: string` |
| **地址结构** | 省-市-区-街道 | **国家-州/省-市-邮编-街道**（分层对象） |
| **状态码** | 国内12个 | **扩展至25+个**，增加清关/航空/海运/退运状态 |
| **语言支持** | 中文为主 | **description对象**，支持zh/en/ja等多语言 |
| **重量字段** | `weight: number` | `weight: { value: number, unit: "KGS"\|"LBS" }` |
| **子包裹** | 无 | **增加`pieces`数组**，支持piece-level tracking |
| **运输工具** | 无 | **增加`transport`对象**（mode/carrier/vesselNumber） |
| **海关信息** | 无 | **增加`customs`对象**（declarationNumber/duty/status） |
| **货币** | 无 | **增加`currency`字段**（ISO 4217，3位字母） |
| **国家代码** | 无 | **所有地址必须含`countryCode`**（ISO 3166-1 alpha-2） |

### 9.3 实施建议

| 阶段 | 行动 | 产出 | 时间 |
|:---|:---|:---|:---:|
| **Phase 0** | 基于国际兼容版数据模型，发布OLTS v0.1草案 | JSON Schema + OpenAPI 3.0 | 2周 |
| **Phase 1** | 编写国内6大快递+国际3大快递（DHL/UPS/USPS）的Python适配器 | 开源代码库 | 1个月 |

### 9.4 风险控制

| 风险 | 可能性 | 影响 | 对策 |
|:---|:---:|:---:|:---|
| 国际字段过多，国内用户觉得复杂 | 高 | 中 | 国内场景字段可为空，不强制填写 |
| 各国海关状态不统一 | 高 | 高 | 定义标准状态码，提供各国映射表 |
| 多语言翻译维护成本高 | 中 | 中 | 采用社区众包+机器翻译 |
| 时区处理增加开发复杂度 | 中 | 低 | 提供标准时区转换库 |
| 国际快递公司API变更频繁 | 中 | 中 | 建立自动化监控+适配器热更新机制 |
